# DATA HEALTH（データ健全性）

## 目的

分析値を見る前に、取込状態・未反映データ・日付カバレッジを確認する読み取り専用の監視画面です。確定、再解析、Alias作成、Cast作成、status変更は行いません。

## 対象範囲

- CTI：春日部・越谷・野田
- Town：春日部・越谷
- Heaven：春日部のみ

対象外店舗は欠損やエラーとして扱いません。Heavenは累積CSVを再投入する運用のため、毎日データがないことだけでは欠損判定しません。

## 品質スコア

基礎点100点から次を減点し、0点未満は0点とします。

- PREVIEW_READY：1件につき8点（最大32）
- FAILED：1件につき10点（最大30）
- WAITING_FOR_CAST_LINK：1件につき3点（最大15）
- OPEN ERROR：1件につき2点（最大20）
- OPEN WARNING：1件につき0.2点（最大10）
- CTIの日付欠損：1日につき5点（最大25）

これはデータの正しさを保証する点数ではなく、取込状態を確認する運用指標です。意図的保留、SKIP、重複終了済み、対象外店舗は重大度を分けます。

## CTI売上影響額

未確定Batchのpreviewから、SKIPPED・ERROR・未紐付け・算出不能行を除外します。`businessDate + storeId + castId`の既存自然キーがある場合は、既存値との差分だけを影響額とします。確定済み重複Batchは二重計上しません。

## 重大度

- 要対応：CTI未確定で売上影響あり、FAILED、OPEN ERRORなど
- 注意：Town/Heavenの未確定、WAITING、警告
- 情報：重複終了済み、管理者判断済みの除外

## 運用フロー

1. HOMEのDATA HEALTHバナーで状態を確認
2. DATA HEALTHでBatch、影響額、日付カバレッジを確認
3. 詳細リンクから既存の取込詳細画面へ移動
4. 管理者が内容を確認してから、既存の確定・再解析フローを実行

## 既知の制約

- previewが欠落しているBatchの影響額は算出不可
- 管理表側の追加項目や外部帳票との差分は算出対象外
- Heavenの累積値は日次欠損として判定しない
- スコアは経営判断の絶対基準ではない
