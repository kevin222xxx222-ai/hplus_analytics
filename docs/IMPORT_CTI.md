# CTI女子別レポート取込仕様

更新日: 2026-07-14

## 対応範囲

Phase 2では日次ファイルだけを `cti_cast_daily` へ確定できます。当月累計・月次確定・UNKNOWNは、期間合計を日次へ誤配賦しないため検証とプレビューのみです。実ファイルの日別内訳が確認できていないため方式Aを採用しました。

## 処理フロー

1. ADMIN権限と同一オリジンを検証
2. サイズ、拡張子、MIME、ZIPマジックバイトを検証
3. SHA-256を算出し、非公開永続領域へUUID名で保存
4. `ImportBatch`をVALIDATINGで作成
5. 対象シート・ヘッダー・列を検出
6. 除外行、値、指標整合性を検証
7. 対象期間を考慮して内部キャストを解決
8. プレビューJSONとImportErrorを保存
9. 管理者が未紐付けを処理
10. 日次のみ確定し、`CtiCastDaily`をupsert
11. ImportBatchの件数と状態を更新

## 対象シート

- 若妻淫乱倶楽部春日部店 → KASUKABE
- 若妻淫乱倶楽部越谷店 → KOSHIGAYA
- 若妻淫乱倶楽部野田店 → NODA

全店舗シートは使用しません。1〜2店舗が不足しても存在する店舗をプレビューし、不足をWARNING保存します。3店舗すべてない場合はFAILEDです。

## ヘッダー検出

各シート先頭50行を走査し、明示的な女子名系ヘッダーがあるか、A列を安全にキャスト名列と推定でき、かつ主要11列のうち8列以上が一致する行を候補にします。一致数が最大の行をヘッダーとし、列順ではなく正規化した列名でマッピングします。主要11列は、出勤数/出勤日数、本指名数、写真指名数、フリー数、予約数、キャンセル数、成約数、女子報酬、利益、出勤時間、料金です。`新規数`は条件に含めません。

A列の安全推定は対象3店舗シートに限り、ヘッダー候補行のA列が空欄、主要列が8列以上一致、後続最大50行のA列に除外行でない文字列名が2件以上存在し、文字列名が数値より多く、有効値の50%以上を占める場合にのみ成立します。成立時は内部で仮想「女子名」列とし、診断画面へ「A1は空欄ですが、データ内容からキャスト名列として認識しました」と表示します。空欄だけを根拠には推定しません。

未知列はWARNING、必須列欠損はERRORです。`新規成約数` / `リピート成約数` は任意列で、欠損時もヘッダー検出を成功させ、値をnull、列単位のWARNINGとします。

実運用74列は `ADOPTED`、`FUTURE_CANDIDATE`、`INTENTIONALLY_UNUSED` の列カタログで管理します。後二者も既知列として認識するためUNKNOWN_COLUMNSにはせず、現時点では型と負数だけを検証して保存しません。本当に未定義の列だけを列ごとのUNKNOWN_COLUMNSとし、店舗、シート、元列名、列番号、ヘッダー行番号を警告本文・raw_data・ImportBatch.detected_columnsへ保持します。

管理画面の取込詳細には、対象シートごとにヘッダー診断を表示します。A〜Zの先頭最大50行、既知列と一致した候補行、一致列数、一致列、必須不足列を確認できます。`HEADER_NOT_FOUND`時は診断を自動展開し、先頭30行を強調表示します。診断機能は正式判定後も維持します。既存FAILEDバッチは保存済み元XLSXから新しいImportBatchとして再解析できます。

## 除外理由コード

- `ANNOUNCEMENT_ROW`: 指定された周知・引継ぎ3名称
- `EMPTY_ROW`: キャスト名が空
- `TOTAL_ROW`: 合計、総合計、小計、計
- `REPEATED_HEADER`: 女子名等のヘッダー繰り返し

Unicode NFKC、前後空白、全角/半角スペース、`＆`/`&`を比較用に正規化します。曖昧な部分一致では除外しません。

## キャスト解決順

1. 対象店舗・期間内のCTI Alias原文一致
2. 店舗共通・期間内のCTI Alias原文一致
3. 対象店舗・期間内のCTI Alias正規化名一致
4. 店舗共通・期間内のCTI Alias正規化名一致
5. 対象日の在籍期間内Castの正規化名が一意に一致
6. 不一致はUNMATCHED、複数候補はAMBIGUOUS

ひらがな/カタカナ、漢字/読み、類似名、編集距離では一致させません。Castの `started_on/ended_on` とAliasの `valid_from/valid_to` を必ず確認します。

## 数値・時間変換

- 金額/整数: Excel数値、カンマ、円、￥、半角/全角を処理
- 出勤時間: 実CTIのGeneral形式数値を10進時間として分換算。Excel時刻書式のシリアル、`H:MM`、`H時間M分`、`M分`にも対応
- 空欄、ハイフン、不正文字は黙って0にせずERROR
- 数式はExcelファイル内の計算済み`result`だけを読む。式は実行しない
- 異常な外れ値はWARNING
- 件数と出勤時間の負数は `NEGATIVE_VALUE`（ERROR）。補正、費用、料金、報酬、利益等のMONEY列は負数を許容し、警告しない
- 負数警告には店舗、キャスト名、シート、行、列、元値、負数許可、理由を保持

## 指標の採用値

- `service_count` = 予約数 − キャンセル数
- `contract_count` = 本指名数 + 写真指名数 + フリー数
- `payout_after_reward_amount` = 料金 − 女子報酬
- `cti_profit_amount` = CTI「利益」原値

CTIに接客数・成約数列がある場合は `source_service_count` / `source_contract_count` に原値を保持します。再計算値との差はWARNINGにし、原値を上書きしません。

`new_count` は「新規成約数」、`repeat_count` は「リピート成約数」のみを採用します。「リピート数」は予約系の別指標として主要項目へ流用しません。

## 重複・再取込

- 同一SHA-256で完了済み履歴がある場合はWARNING、明示的再処理が必要
- 日次実績キーは `business_date + store_id + cast_id`
- 修正版は最新値でupsertし、`import_batch_id`を更新
- 新規/更新件数は確定前の存在確認で分離
- 更新差分はImportBatch.metadataへ保存
- 修正版から消えた行は自動削除しない

## トランザクション

日次3店舗の1ファイルを1トランザクションとし、CtiCastDaily upsert、部分取込警告、ImportBatch件数/状態を同時更新します。タイムアウトは60秒です。将来件数が増えた場合は、ImportBatchを親にした安全なチャンク方式へ変更します。

## ファイル保存

- `UPLOAD_DIR`（Dockerでは`/app/data/uploads`）
- Docker永続ボリューム `hplus_analytics_uploads`
- 内部名 `{ImportBatch.id}.xlsx`
- プレビュー `{ImportBatch.id}.preview.json`
- DBへバイナリ保存しない
- `public/`へ保存しない
- ダウンロードはADMINのみ
- 上限は `MAX_UPLOAD_SIZE_MB`（既定20MB）
