# Marketing Management OS

## 目的

店舗の絶対売上だけでなく、出勤日・出勤時間あたりの効率、集客から予約・成約・本指名までの流れを比較し、店長が次の確認行動を選ぶための読み取り専用ダッシュボードです。診断・提案は因果関係を断定せず「候補」「参考値」と表示します。

## 画面構成

`/` を店舗運営・マーケティング司令塔とし、当月・前月・過去3か月・過去6か月・任意期間、春日部・越谷・野田（CTI補助）を切り替えます。

- 着地・目標サマリー
- 不足要因候補
- 店舗全体の健康状態
- 隠れエース候補
- 埋もれキャスト候補
- 出勤増加候補
- ボトルネック候補
- 詳細分析画面への導線

## 指標定義

CTIは `CtiCastDaily`、Townは `TownCastDaily`、Heavenは `HeavenCastDaily` を期間内で集計します。DAILY_EVENTは合計、SNAPSHOTは最終値と期間増減です。媒体に値がない場合は0ではなく算出不能（画面上は「—」）とします。

主な効率指標は売上/出勤日、売上/出勤時間、女子報酬/出勤日、女子報酬/出勤時間、成約/出勤日、PV/出勤日、本指名/接客、本指名率、成約/予約、成約/Town UU、成約/Heaven PAGE_ACCESS、女子報酬/接客、売上/接客です。分母0はnullです。

CTI同日複数店舗出勤は全体の出勤日数では1日として扱います。Town PVとHeaven PAGE_ACCESSは顧客単位で対応しないため、合算する場合も「参考合計」です。

## 比較基準と候補ルール

店舗中央値と上位25%を基準にします。隠れエース候補は、出勤日数が中央値以下、売上/出勤日と女子報酬/出勤時間が上位25%、本指名率・PV/出勤日が平均以上のキャストです。埋もれ候補は、出勤日数が中央値以上で売上/出勤日とPV/出勤日が中央値・平均を下回るキャストです。

ボトルネック候補は、集客不足、閲覧後転換不足、成約不足、再指名不足のルールに該当する場合に根拠指標とともに表示します。推奨施策は、露出・プロフィール、日記、ミテネ、オキニトーク、予約導線、出勤相談などのルールベース表示です。

## 着地予測

目標が設定されている場合、経過日あたりの売上を残り日数へ単純外挿します。曜日補正や出勤予定は、設定データが存在しないため初版では未使用です。予測はAIではなく参考値です。目標はDBへ追加せず、`DASHBOARD_MONTHLY_SALES_TARGET` などの環境変数が設定されている場合のみ表示します。

## 既知の制約・将来拡張

# Navigator・目標管理連携

HOMEは全体を初期表示し、`MonthlyGoal` の全体目標を達成率・着地予測へ利用する。店舗目標が未設定でも全体目標を按分しない。詳細な導線は `docs/ANALYTICS_NAVIGATOR.md` を参照。
# DATA HEALTH連携

HOME上部のDATA HEALTHバナーは、確定済み実績と未確定影響見込みを分けて表示します。未確定額をHOME売上へ加算せず、`/data-health`へ誘導します。
